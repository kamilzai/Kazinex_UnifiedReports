# Admin UI Specification

**Date:** December 11, 2025  
**Purpose:** Admin interface for managing report designs, sections, and structures  
**Users:** System administrators

---

## Overview

The admin UI allows authorized users to:
- Create and manage **Report Designs** (templates)
- Define **Sections** (tabs within a report)
- Configure **Structures** (columns within each section)
- Manage **Lookup Options** (dropdown values)
- Preview report layouts before publishing

**UI Framework:** Ant Design 5.x  
**Layout:** Sidebar navigation + main content area  
**URL Structure:** `/admin/*`

## User Flows

### Flow 1: Create New Report Design

1. Admin clicks "+ New Design" from designs list
2. Fills basic information (name, description)
3. Clicks "Add Section" to add first tab
4. Configures section (name, enable grouping)
5. Clicks "Add Column" to add columns
6. For each column:
   - Sets field name, display name
   - Selects type (text, number, etc.)
   - Configures type-specific settings (lookup options, formulas, etc.)
   - Sets column size and validation
7. Repeats steps 3-6 for additional sections
8. Clicks "Preview" to see layout
9. Clicks "Save Design" to publish

### Flow 2: Clone Existing Design

1. Admin clicks "Clone" on existing design
2. System creates copy with " (Copy)" suffix
3. Admin edits as needed
4. Saves as new design

### Flow 3: Manage Lookup Options

1. Admin edits column of type "Lookup"
2. Clicks "Manage Options"
3. Modal opens with existing options
4. Admin can:
   - Add new options
   - Edit existing options
   - Reorder options (drag and drop)
   - Activate/deactivate options
5. Clicks "Save All" to persist changes

## Access Control

### Middleware Protection

```typescript
// src/middleware.ts

export async function middleware(request: NextRequest) {
  const path = request.nextUrl.pathname;
  
  // Protect /admin routes
  if (path.startsWith('/admin')) {
    const session = await getServerSession();
    
    if (!session) {
      return NextResponse.redirect(new URL('/login', request.url));
    }
    
    const userRole = await getUserRole(session.user.id);
    
    if (userRole !== 'admin') {
      return NextResponse.json(
        { error: 'Unauthorized' },
        { status: 403 }
      );
    }
  }
  
  return NextResponse.next();
}
```
